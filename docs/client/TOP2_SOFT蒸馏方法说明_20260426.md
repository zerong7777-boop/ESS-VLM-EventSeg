# Top-2 Soft 蒸馏方法说明

## 方法动机

原始 `ESS + FC-CLIP` 路线使用 FC-CLIP 为 DDD17 训练样本生成离线 dense 伪标签，再把这些 hard label 作为事件分支的额外监督信号。

这种方式已经能把 DDD17 事件语义分割 mIoU 从官方 baseline 的 `53.05` 提升到 `56.36`。但是 hard label 只保留了教师模型最确定的一个类别，丢掉了教师模型对第二候选类别的判断。

在事件相机语义分割中，很多像素本身存在模糊性，例如边界区域、远处小目标、运动模糊区域。只用 top-1 hard label 可能会把教师的不确定性变成过强监督。因此本版增加了 Top-2 Soft 蒸馏：保留 FC-CLIP 对每个像素最可能的两个类别及其概率，用较软的分布约束 ESS 事件分支。

## 方法形式

对每个训练像素，FC-CLIP 离线导出：

- `top1_id`：教师模型概率最高的类别；
- `top2_id`：教师模型概率第二高的类别；
- `p1`：top-1 类别概率；
- `p2`：top-2 类别概率。

训练时，ESS 事件分支仍然保留原有 hard pseudolabel CE，同时额外加入 top-2 soft distillation loss：

```text
L = L_task + w_hard * L_hard_pseudolabel + w_soft * L_top2_soft
```

其中 `L_top2_soft` 只在 top-1 和 top-2 两个教师类别上计算：

```text
L_top2_soft = - w1 * log P_student(top1_id) - w2 * log P_student(top2_id)
```

`w1` 和 `w2` 由 `p1`、`p2` 归一化得到。

本版优化后的关键设置为：

```yaml
use_top2_soft_distillation: true
weight_top2_soft_distillation: 0.015
top2_soft_temperature: 1.0
top2_soft_min_p1: 0.0
top2_soft_min_margin: 0.0
```

## 本版结果

本版最优结果来自：

```text
config/settings_DDD17_fcclip_top2_soft_best_w0015.yaml
checkpoint: Epoch_15.pt
```

验证结果：

| Epoch | mIoU | Acc |
| ---: | ---: | ---: |
| 13 | 52.0516 | 86.5858 |
| 14 | 53.0631 | 89.1762 |
| 15 | 56.8124 | 89.5776 |
| 16 | 54.3424 | 89.3049 |
| 17 | 55.7079 | 89.7986 |
| 18 | 56.1059 | 89.6507 |

结论：

- best checkpoint 为 `Epoch 15`；
- best mIoU 为 `56.8124`；
- 相对官方 ESS baseline `53.05`，提升 `+3.7624`；
- 相对上一版 top-2 soft 最强 checkpoint `56.7029`，提升 `+0.1095`；
- 该方法属于 checkpoint-sensitive 增益，答辩或汇报时应明确采用 best checkpoint 口径，而不是 final epoch 口径。

## 对外表述建议

可以表述为：

> 在 DDD17 数据集上，本文在 ESS 框架基础上引入 FC-CLIP 视觉语言模型生成的离线伪标签，并进一步设计 Top-2 Soft 蒸馏约束，保留教师模型对前两类候选语义的概率信息。该方法在 best checkpoint 上将事件语义分割 mIoU 从官方 baseline 的 53.05 提升到 56.8124，验证了视觉语言模型辅助事件相机语义分割的有效性。
